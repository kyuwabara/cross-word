# クロスワードパズル アプリケーション

5歳児のひらがな学習用クロスワードパズル生成 Web アプリ。GitHub Pages で配信する PWA。

## 技術スタック

- **フロントエンド**: バニラ HTML / CSS / JavaScript（フレームワーク不使用）
- **オフライン対応**: Service Worker（cache-first）+ Web App Manifest
- **ホスティング**: GitHub Pages（静的ファイルのみ、サーバーサイドなし）
- **クロスワード生成**: ブラウザ内の JavaScript による貪欲配置アルゴリズム

## ファイル構成

- `index.html` - エントリポイント（Service Worker 登録スクリプト含む）
- `manifest.webmanifest` - PWA マニフェスト
- `sw.js` - Service Worker（cache-first、スコープ確保のためリポジトリルートに配置）
- `.nojekyll` - GitHub Pages の Jekyll 処理を無効化
- `static/style.css` - スタイルシート（タブレット用レイアウト + 印刷用スタイル）
- `static/script.js` - UI ロジック（セル操作・キーボード・こたえあわせ）
- `static/crossword.js` - クロスワード生成アルゴリズム（`generateCrossword(w, h)` をグローバルに公開）
- `static/words.js` - 5歳児向け単語リスト（`WORDS` 配列、ひらがな + ヒント）
- `static/icon.svg` - PWA アイコン

静的アセット参照は `?v=3` のようなクエリでバージョニングしている（`index.html` 内）。中身を変えたらクエリと `sw.js` の `CACHE_NAME` を揃えて上げる。

## ローカル確認

```bash
python3 -m http.server 8080
```

ブラウザから `http://localhost:8080/` にアクセス。

## デプロイ

main ブランチへ push すると GitHub Pages から自動配信される。
公開 URL: https://kyuwabara.github.io/cross-word/

**重要**: 配信内容を変えたら `sw.js` の `CACHE_NAME`（例: `crossword-v4` → `crossword-v5`）を必ず上げる。上げないとタブレット側の Service Worker が古いキャッシュを使い続け、更新が反映されない。

## 仕様

### クロスワードルール
- ひらがなのみ使用（簡単な外来語もひらがな表記）
- 小さい仮名（ぁぃぅぇぉっゃゅょ）は大きい仮名（あいうえおつやゆよ）に変換して配置（日本語クロスワードの慣例）
- 各単語は 2 文字以上
- 最初の単語以外は既存の単語と必ず交差する
- 単語の前後（先頭の前・末尾の後ろ）は黒マスまたはグリッドの端
- 意図しない並行単語（配置したつもりのない隣接文字列）が生まれないように検査

### 画面仕様
- タブレット操作に最適化（タッチ対応、大きなマス・ボタン）
- ページ読み込み時に自動でパズル生成
- グリッドサイズ 5×5 〜 15×15 で指定可能
- **ひらがなオンスクリーンキーボード**: 五十音表レイアウト（右から縦に「あいうえお」列）、清音・濁音・半濁音すべて入力可能、キーボード下部に「けす」ボタン
- **セル選択**: タップで選択、同じセルを再タップで縦横切り替え。選択中の単語全体をハイライトし、対応するヒントを自動スクロールで強調表示
- **ヒントタップ**: 対応するセルの先頭にジャンプ
- **こたえあわせ**: 正解セルは緑、不正解セルは赤で表示。不正解セルには正解の文字を青字で重ね表示。結果メッセージ（全問正解 / 部分正解）を表示
- **ぜんぶけす**: 入力をすべてリセット
- **あたらしいパズル**: 現在のグリッドサイズで新しいパズルを生成
- **印刷モード (`いんさつ`)**: キーボード・ボタン・こたえ表示を隠して、空のグリッド＋ヒントのみを印刷（`@media print` で制御）

### オフライン挙動
- 初回オンラインアクセス時に Service Worker が全アセットを precache
- 以降は cache-first で応答、ネットワーク不通でもアプリが起動する
- ホーム画面に追加すると standalone 表示で起動（`manifest.webmanifest` の `display: "standalone"`）
